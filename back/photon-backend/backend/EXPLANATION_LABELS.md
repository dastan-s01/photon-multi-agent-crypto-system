# Объяснение генерации меток для обучения модели

## 📊 Ситуация: у нас 720 свечей (hourly данные за 30 дней)

### Шаг 1: Подготовка данных

```
Свеча 1:  [10:00] Цена = $90,000
Свеча 2:  [11:00] Цена = $90,500
Свеча 3:  [12:00] Цена = $91,000
...
Свеча 720: [через 30 дней] Цена = $95,000
```

### Шаг 2: Для каждой свечи извлекаем фичи (технические индикаторы)

**Пример для свечи #100:**
- Цена закрытия: $92,000
- Объем: 1000 BTC
- RSI: 55 (нейтральный)
- MACD: +50 (бычий сигнал)
- SMA10: $91,500
- SMA20: $91,000
- И т.д. (всего 13 фичей)

### Шаг 3: Генерация метки (label) - САМОЕ ВАЖНОЕ!

**Для свечи #100 мы смотрим на цену через 6 часов вперед:**

```
Свеча #100: [10:00] Цена = $92,000  ← ТЕКУЩАЯ (для которой генерируем метку)
Свеча #101: [11:00] Цена = $92,100
Свеча #102: [12:00] Цена = $92,200
Свеча #103: [13:00] Цена = $92,300
Свеча #104: [14:00] Цена = $92,400
Свеча #105: [15:00] Цена = $92,500
Свеча #106: [16:00] Цена = $92,800  ← СМОТРИМ СЮДА (через 6 часов)
```

**Вычисляем:**
1. Изменение цены: `($92,800 - $92,000) / $92,000 = 0.87%`
2. Комиссия: `0.1% × 2 = 0.2%` (вход + выход)
3. Чистая прибыль: `0.87% - 0.2% = 0.67%`
4. Минимальный порог: `0.5%`

**Результат:** `0.67% > 0.5%` → **МЕТКА = BUY (2)**

**Почему BUY?** Потому что если бы мы купили на свече #100 по $92,000 и продали через 6 часов по $92,800, мы бы получили прибыль 0.67% (после комиссии).

---

### Шаг 4: Другой пример - SELL

**Для свечи #200:**
```
Свеча #200: [10:00] Цена = $94,000  ← ТЕКУЩАЯ
...
Свеча #206: [16:00] Цена = $93,400  ← ЧЕРЕЗ 6 ЧАСОВ (упала!)
```

**Вычисляем:**
1. Изменение цены: `($93,400 - $94,000) / $94,000 = -0.64%`
2. Комиссия: `0.2%`
3. Чистая прибыль: `0.64% - 0.2% = 0.44%` (но это убыток, так как -0.64%)
4. Для SELL: цена упала на 0.64%, что больше порога 0.5%

**Результат:** `-0.64% < -0.5%` → **МЕТКА = SELL (0)**

**Почему SELL?** Потому что цена упала достаточно, чтобы продать (или открыть шорт).

---

### Шаг 5: Пример - HOLD

**Для свечи #300:**
```
Свеча #300: [10:00] Цена = $93,000  ← ТЕКУЩАЯ
...
Свеча #306: [16:00] Цена = $93,200  ← ЧЕРЕЗ 6 ЧАСОВ (выросла немного)
```

**Вычисляем:**
1. Изменение цены: `($93,200 - $93,000) / $93,000 = 0.22%`
2. Комиссия: `0.2%`
3. Чистая прибыль: `0.22% - 0.2% = 0.02%` (почти ноль!)
4. Минимальный порог: `0.5%`

**Результат:** `0.22% < 0.5%` → **МЕТКА = HOLD (1)**

**Почему HOLD?** Потому что движение слишком маленькое, не покрывает комиссию и минимальный порог прибыли.

## 📈 Итоговая таблица для 720 свечей

```
Свеча #1:   Фичи [13 значений] → Метка: BUY   (смотрим на свечу #7)
Свеча #2:   Фичи [13 значений] → Метка: HOLD  (смотрим на свечу #8)
Свеча #3:   Фичи [13 значений] → Метка: SELL  (смотрим на свечу #9)
...
Свеча #714: Фичи [13 значений] → Метка: BUY   (смотрим на свечу #720)
Свеча #715: Фичи [13 значений] → Метка: ???   (НЕТ! Недостаточно данных вперед)
...
Свеча #720: Фичи [13 значений] → Метка: ???   (НЕТ! Это последняя свеча)
```

**Итого:** Из 720 свечей мы можем создать **714 samples** для обучения (720 - 6 = 714)

---

## 🎯 Почему это правильно?

### ❌ СТАРАЯ ЛОГИКА (неправильная):
- Смотрели только на следующую свечу (1 час)
- Использовали индикаторы в логике меток (data leakage)
- Не учитывали комиссию

**Проблема:** Модель училась на "легких" примерах, которые не отражают реальную торговлю.

### ✅ НОВАЯ ЛОГИКА (правильная):
- Смотрим на 6 часов вперед (более реалистично)
- Метки основаны ТОЛЬКО на будущей цене (без индикаторов)
- Учитываем комиссию и минимальную прибыль

**Преимущество:** Модель учится на реалистичных сценариях, которые отражают реальную торговлю.

## 🔢 Математика в цифрах

**Для свечи #100:**
- Текущая цена: $92,000
- Цена через 6 часов: $92,800
- Изменение: `(92,800 - 92,000) / 92,000 × 100 = 0.87%`
- Комиссия: `0.1% + 0.1% = 0.2%`
- Чистая прибыль: `0.87% - 0.2% = 0.67%`
- Порог: `0.5%`
- **Решение:** `0.67% > 0.5%` → **BUY** ✅
